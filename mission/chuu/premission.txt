[1] 컨테이너 기술
컨테이너 기술은 호스트 시스템의 리소스를 이용하여 애플리케이션을 패키징하고 실행하는 방법이다. 또한 호스트 운영 체제를 공유하고 컨테이너 엔진을 사용하여 각 컨테이너의 격리 및 리스소를 관리한다.

[2] 도커
도커는 애플리케이션과 denpendency를 경량 컨테이너로 패키징할 수 있게 하는 컨테이너 플랫폼이다. 도커 내 컨테이너는 서로 격리되어 있고 Docker가 설치된 모든 시스템에서 실행할 수 있다.

[3] 도커 파일
Dockerfile : Docker에 Docker 이미지 빌드 방법을 알려주는 지침이 포함된 스크립트

[4] 도커 이미지
Docker image : 애플리케이션을 실행하는 데 필요한 모든 파일, 종속성 및 구성을 포함하는 읽기 전용 템플릿. Dockerfile로부터 빌드되며 Docker Registry에 저장 및 배포 가능. Docker image는 이식 가능하며 Docker가 설치된 모든 시스템에서 실행 가능.

[5] 도커 컨테이너
Docker container : Docker 이미지의 실행가능한(runnable) 인스턴스. Docker 이미지으로부터 생성되며 애플리케이션 코드, 종속성 및 구성을 포함한다. 컨테이너는 가볍고 휴대 가능하며 호스트 시스템 및 기타 컨테이너와 격리되어 있다.

[6] 관계
Dockerfile은 Docker 이미지를 빌드하는 데 사용되고,
Docker 이미지는 Docker 컨테이너를 만드는 데 사용되며,
Docker 컨테이너는 Docker 이미지의 실행 가능한 인스턴스이다.
